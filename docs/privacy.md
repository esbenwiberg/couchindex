# CouchIndex Privacy Policy

**Effective date:** July 13, 2026

CouchIndex is a Google TV application that combines catalogue and availability data
for streaming services selected by the user. It does not require a CouchIndex account
and does not include advertising or analytics SDKs.

The developer responsible for CouchIndex can be contacted using the address in the
Contact section below.

## Information stored on the device

CouchIndex stores the following app-private information locally:

- selected streaming subscriptions
- watchlist membership
- recent provider launches and manual watched state
- Like and Dislike feedback
- cached catalogue, artwork and rating data

This information is used only to provide the app's browsing, Continue Watching and
recommendation features. CouchIndex does not upload this personal state to its own
server. App data is excluded from Android cloud backup and device-to-device transfer.
It can be removed by clearing app storage or uninstalling CouchIndex.

## Network services

CouchIndex connects directly to:

- TMDb for catalogue metadata, artwork, identifiers and regional provider availability
- IMDb's public non-commercial dataset host for title ratings
- streaming-provider apps, TMDb watch pages or Google Play only when the user chooses a
  handoff or installation action

Those external services receive normal internet request information such as the
device's IP address and request metadata and process it under their own policies.
CouchIndex does not send them the local watchlist, watched history or Like and Dislike
feedback.

Availability data is supplied by JustWatch through TMDb. This product uses the TMDb API
but is not endorsed or certified by TMDb.

## Sharing and sale

CouchIndex does not sell personal information. It does not share local personal state
with advertisers, data brokers or analytics providers.

## Children

CouchIndex is not designed specifically for children and does not knowingly collect
personal information from children through an account or submission form.

## Changes

Material changes to this policy must be published before an app update that introduces
the corresponding data practice.
